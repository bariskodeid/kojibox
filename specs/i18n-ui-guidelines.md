# i18n Guidelines (UI General)

This document defines translation rules for general UI copy.

Rules:
- Keep labels short (1-3 words).
- Prefer verbs for actions (Start, Stop, Restart).
- Avoid abbreviations unless common (DB, TLS).
- Use consistent service names across UI.
- Keep placeholders clear and singular (no nested placeholders).

Keys (examples):
- ui.nav.dashboard
- ui.nav.projects
- ui.nav.logs
- ui.action.start
- ui.action.stop
- ui.action.restart
- ui.action.open_logs
- ui.status.running
- ui.status.stopped
- ui.status.error
- ui.project.add
- ui.project.edit

Placeholders:
- {service}
- {project}
- {port}
